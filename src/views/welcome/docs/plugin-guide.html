Membuat Plugin Pertama Anda

Simaju Core dirancang dengan arsitektur modular yang memungkinkan Anda memperluas fungsionalitas tanpa mengubah kode inti framework. Setiap plugin adalah unit mandiri yang memiliki rute, model, dan kontroler sendiri.

Struktur Folder Plugin

Semua plugin diletakkan di dalam folder src/plugins/. Berikut adalah struktur standar yang direkomendasikan:

my-plugin/
├── plugin.json // Metadata plugin
├── routes.js // Definisi rute API
├── controller.js // Logika bisnis
├── model.js // Skema/Akses database
├── migrations/ // Skrip tabel database
└── index.js // (Opsional) Entry point hook

1. Definisi Metadata (plugin.json)

File ini wajib ada untuk mengidentifikasi plugin Anda di Dashboard Admin.

{
  "name": "My Awesome Plugin",
  "version": "1.0.0",
  "description": "Plugin untuk fitur X",
  "author": "Nama Anda"
}

2. Definisi Rute (routes.js)

Plugin akan otomatis didaftarkan pada endpoint /api/plugins/[folder-name].

const express = require('express');
const router = express.Router();
const Controller = require('./controller');

router.get('/', Controller.index);

module.exports = router;

3. Akses Database (model.js)

Anda dapat menggunakan Core Database wrapper untuk akses yang mudah.

const DB = require('../../database');

class MyPluginModel {
  static async getAll() {
    return await DB.table('plugin_table').select('*');
  }
}

module.exports = MyPluginModel;

💡 Tips Profesional

Selalu buat migrasi database di dalam folder migrations/ agar pengguna bisa memasang plugin Anda dengan perintah ./simaju plugin:migrate.
